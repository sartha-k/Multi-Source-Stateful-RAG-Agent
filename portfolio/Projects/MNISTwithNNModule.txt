MNIST Digit Classification with nn.Module — PyTorch
End-to-end implementation of a neural network trained on the MNIST handwritten digits dataset using PyTorch's nn.Module API. Achieved 87.52% accuracy on the test set.

What This Notebook Covers
Loading and preprocessing MNIST data from a CSV file using Pandas
Converting a Pandas DataFrame directly into PyTorch tensors
Splitting data into train/test sets using train_test_split
Building a custom neural network class with nn.Module
Training with NLLLoss + explicit log_softmax
Evaluating accuracy using argmax on test predictions
Dataset
Source: Kaggle Digit Recognizer Competition
Format: CSV — 42,000 samples, 785 columns (1 label + 784 pixel features)
Classes: 10 (digits 0–9)
Pixels: Already normalized (values 0 or 1)

Input (batch_size, 784)

Model:

class SimpleNN(nn.Module):
    def __init__(self, num_feature):
        super().__init__()
        self.linear = nn.Linear(num_feature, 10)

    def forward(self, x):
        return self.linear(x)  # raw logits
Training Loop:

for epoch in range(epochs):
    y_pred    = model(X_train)
    log_probs = torch.log_softmax(y_pred, dim=1)
    loss      = lossFunc(log_probs, y_train)
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

Lessons Learned
pd.read_csv() needs a file path, not a directory path
torch.from_numpy() requires NumPy array, not DataFrame — use .values
NLLLoss requires log_softmax output, not raw logits
loss.backward without () silently does nothing — always use loss.backward()
High learning rate causes loss to spike — reduce or use Adam optimizer
argmax is needed to convert model output to class predictions
